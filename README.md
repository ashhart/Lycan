# Lycan

A programming language designed for AI to generate and machines to execute.

Most code is written for humans and interpreted by machines.
Lycan is written for machines and generated by AI.

Traditional app code asks an LLM to infer intent from files, names, classes, comments, frameworks, and side effects. Lycan removes that layer. A Lycan program is compact, explicit, graph-shaped, policy-bounded, and executable.

## What Lycan is

- `.lycs` is the source language (S-expression syntax)
- `.lyc` is the compiled graph binary (weighted neural program)
- `.lycap` is the capsule exchange format (program + policy + manifest + journal)
- The Lycan runtime executes graphs directly, learns from outcomes, and evolves under verification

The first killer use case is adaptive decisions. But the substrate is broader: Lycan is a general-purpose execution format for AI-generated computation.

## Syntax primer

```lisp
;; Values
42              ;; integer
3.14            ;; float
"hello"         ;; string
true / false    ;; boolean
null            ;; null
(A 1 2 3)       ;; array

;; Bindings
($ x 42)        ;; immutable
($! x 0)        ;; mutable
(= x 10)        ;; reassign

;; Functions
(F add (a b) (+ a b))
(!p (add 3 4))         ;; prints 7

;; Control flow
(? (> x 10) "big" "small")     ;; if/else
(W (< i 10) body...)           ;; while
(each x collection body...)    ;; for-each
(B expr...)                    ;; block

;; Collections
(A 10 20 30)           ;; array literal
(I arr 1)              ;; index access
(.. 1 5)               ;; range

;; Strategy nodes (where programs learn)
($ result (strategy
  (fast_method args)
  (accurate_method args)
  (experimental_method args)))

;; Capabilities (Rust-native kernels)
(!cap "stats.mean" data)
(!cap "http.get" "https://api.example.com/data")
(!cap "file.readText" "config.json")
(!cap "runtime.inputGet" "request.body.symbol")

;; Output
(!p "hello from lycan")
```

## Strategy nodes â€” where programs learn

Multiple implementations compete. The runtime explores, measures, validates correctness, and persists learned weights:

```lisp
(F sum_loop (n)
  ($! total 0) ($! i 1)
  (W (<= i n) (= total (+ total i)) (= i (+ i 1)))
  total)

(F sum_formula (n)
  (/ (* n (+ n 1)) 2))

($ result (strategy (sum_loop 5000) (sum_formula 5000)))
```

After multiple runs, the formula wins â€” 1477x faster, 99% confidence. Output stays correct.

## Build and run

```bash
cargo build --release

# Run source
lycan program.lycs

# Compile to binary
lycan compile program.lycs

# Execute binary (learns on each run)
lycan program.lyc

# Decision with JSON input
lycan decide program.lyc --input request.json

# Feedback
lycan feedback program.lyc <node> --option <n> --reward <f>

# Autonomous evolution
lycan evolve program.lyc --proposal proposal.json

# Capsule lifecycle
lycan capsule create program.lyc name "intent"
lycan capsule verify name.lycap
lycan capsule run name.lycap
```
